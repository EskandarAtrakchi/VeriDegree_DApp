WHAT WAS THE PROBLEM?
  Error: "Access to fetch at 'https://rpc.sepolia.org/' from origin 
          'http://localhost:3000' has been blocked by CORS policy"
  
  Cause: Browser tried to call blockchain RPC directly
         RPC endpoint blocked requests from browser
         CORS policy prevented the call

WHAT'S THE SOLUTION?
  - Created backend API route: /api/certificates/get-for-address
  - Viewer & Verifier now call this API route instead of RPC directly
  - Backend API queries RPC (server-to-server, no CORS issues)
  - Pinata integration stays the same and works perfectly

DID YOU NEED NEW PINATA KEYS?
  NO! Your original Pinata credentials are perfect
  The issue was NOT with Pinata at all
  It was with how the browser accessed the blockchain
